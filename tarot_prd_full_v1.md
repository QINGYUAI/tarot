## 一、产品概述

### 1.1 一句话定位

专为塔罗零基础小白设计的**引导式 + AI 深度解读**占卜工具，用户无需任何塔罗知识，即可完成一次完整、有意义、被理解的占卜体验。

### 1.2 目标用户

- 对塔罗有兴趣但完全不了解的普通人
- 有占卜需求但不知道该怎么开始
- 想要快速获得有针对性、可操作建议的用户

### 1.3 核心价值主张

让"不知道问什么 → 不知道怎么抽 → 抽到了看不懂 → 看了牌义不知道怎么办"这条障碍链，在一次使用中全部打通。

### 1.4 关键词

引导式 / 小白友好 / AI 深度解读 / 仪式感 / 温暖陪伴

## 二、用户故事与核心场景

### 场景一：迷茫时的快速解答

> 用户：我最近工作很迷茫，想知道接下来该怎么办。

**路径**：选"今日运势" → 单张抽牌 → AI 解读 → 获得方向性建议

### 场景二：重要决策前的深度分析

> 用户：我要不要和 TA 分手，这件事拖了很久了。

**路径**：选"感情"场景 → 凯尔特十字（7张）→ AI 深度解读 → 获得情感建议和行动指引

### 场景三：选 A 还是选 B

> 用户：有两个 offer，我该选哪个？

**路径**：选"三张牌（选 A/B/C）" → 抽牌 → AI 对比分析 → 做出选择参考

### 4.4 洗牌与抽牌

#### 4.4.1 洗牌

- 视觉：卡片叠放随机切换动画
- 可选跳过（按钮："直接开始"）
- 时长：约 5 秒动画

#### 4.4.2 抽牌

**流程**：
1. 所有牌背面朝上显示（牌背统一）
2. 用户点击一张牌 → 翻转动画 → 正面显示
3. 抽够指定数量 → 停止抽牌
4. 显示牌阵确认页

**正位/逆位**：
- 抽出的牌随机正位或逆位
- 逆位牌图片旋转 180° 显示
- 逆位解读与正位不同

### 4.5 AI 深度解读

#### 4.5.1 输入

```json
{
  "userQuestion": "我最近工作很迷茫，该怎么办？",
  "spreadType": "celtic_cross",
  "cards": [
    { "position": 1, "card": "The High Priestess", "isReversed": false },
    { "position": 2, "card": "The Tower", "isReversed": true },
    { "position": 3, "card": "The Empress", "isReversed": false }
    // ... 根据牌阵张数
  ],
  "cardMeanings": {
    "The High Priestess": { "upright": "...", "reversed": "..." },
    // ...
  }
}
```

#### 4.5.2 输出结构

AI 解读统一返回以下四个部分：

**① 整体概述**（50-100字）
> 用温暖易懂的语言总结这段占卜的核心信息，1-2句话说明老天/牌面在告诉你什么。

**② 分位解读**（每位置 80-150字）
> 结合该位置的含义 + 牌的牌义 + 正/逆位，给出针对性解读。
> 语气：像朋友在给你分析，不是老师在讲课。

**③ 行动建议**（3-5条）
> 明确的、可操作的建议，格式：
> - "建议1：..."
> - "建议2：..."

**④ 注意事项**（1-3条）
> 温和的提醒，格式：
> - "⚠️ 需要注意：..."

#### 4.5.3 AI 提示词风格要求

- ✅ 温暖、有同理心
- ✅ 清晰、不玄学（避免过度神秘化语言）
- ✅ 不宿命论（强调选择和行动的力量）
- ✅ 小白友好（术语出现时附带简要解释）
- ❌ 不说"牌面显示..."、"这是命中注定..."
- ❌ 不夸大恐惧或希望

#### 4.5.4 模型配置

**配置文件**（`.env`）：

```env
VITE_AI_MODELS=[{"id":"deepseek","name":"DeepSeek","enabled":true},{"id":"gpt4","name":"GPT-4","enabled":false},{"id":"claude","name":"Claude","enabled":false}]
VITE_DEFAULT_MODEL=deepseek
```

**API 调用方式**：
- 前端调用后端接口（`POST /api/tarot/interpret`）
- 后端从环境变量读取各模型 Key
- 支持流式输出（打字机效果）

---

### 4.6 多模型切换

**UI**：
- 解读生成页顶部有模型选择下拉框
- 显示当前模型名称（如 "DeepSeek"）
- 模型选项由 `.env` 配置，`enabled: true` 的模型才显示

**交互**：
- 默认使用 `VITE_DEFAULT_MODEL`
- 用户切换 → 重新生成当前解读
- 生成中显示 loading 状态

### 5.3 数据模型

```typescript
// types/tarot.ts

// 78张牌
interface TarotCard {
  id: string;           // 'fool', 'magician'...
  name: {
    en: string;
    zh: string;
  };
  arcana: 'major' | 'minor';
  suit?: 'wands' | 'cups' | 'swords' | 'pentacles';
  rank?: string;       // 'ace' | '2'-'10' | 'page' | 'knight' | 'queen' | 'king'
  keywords: {
    upright: string;
    reversed: string;
  };
  meanings: {
    upright: string;
    reversed: string;
  };
  image: string;       // 图片 URL
}

// 牌阵
interface Spread {
  id: string;
  name: {
    en: string;
    zh: string;
  };
  description: string;
  cardCount: number;
  positions: SpreadPosition[];
}

interface SpreadPosition {
  index: number;
  name: string;
  description: string;
}

// 抽出的牌
interface DrawnCard {
  card: TarotCard;
  position: number;
  isReversed: boolean;
}

// 用户占卜记录
interface DivinationRecord {
  id: string;
  question: string;
  spread: Spread;
  cards: DrawnCard[];
  interpretation?: string;
  model: string;
  createdAt: string;
}
```

## 六、视觉与交互规范

### 6.1 视觉风格

**关键词**：神秘感 / 优雅 / 现代 / 温暖 / 不阴森

**配色方案**：

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#1a1035` | 页面背景（深邃紫黑） |
| 副色 | `#2d1b69` | 卡片背景、模块底色 |
| 点缀色 | `#d4af37` | 按钮高亮、标题、装饰线 |
| 强调色 | `#9b7ed9` | 交互反馈、次要高亮 |
| 文字主色 | `#f5f0e8` | 正文（暖白，保护视力） |
| 文字次色 | `#a89cc8` | 说明文字、次要信息 |
| 牌背面 | `#3d2066` | 牌背底色 |
| 正位框 | `#d4af37` | 牌面边框（正位） |
| 逆位框 | `#8b5a2b` | 牌面边框（逆位） |

**字体**：
- 标题：衬线体（如 Noto Serif SC）
- 正文：清晰无衬线（如 Noto Sans SC）

### 6.2 响应式断点

| 设备 | 宽度 |
|------|------|
| 手机 | < 768px |
| 平板 | 768px - 1024px |
| 桌面 | > 1024px |

### 6.3 关键交互动效

| 动效 | 描述 |
|------|------|
| 洗牌动画 | 牌背图片快速随机切换，5秒 |
| 翻牌动画 | 3D 翻转（rotateY），0.5秒 |
| 解读打字机 | 流式文字逐字显示 |
| 页面切换 | 淡入淡出，0.3秒 |

## 七、部署说明

### 7.1 前端部署

```bash
# 构建
npm run build

# 输出目录 dist/
# 部署到服务器任意目录，配置 nginx 指向 dist/
```

### 7.2 环境变量

```env
# .env
VITE_APP_TITLE=塔罗占卜
VITE_AI_MODELS=[{"id":"deepseek","name":"DeepSeek","enabled":true}]
VITE_DEFAULT_MODEL=deepseek
```

### 7.3 后端（AI 代理）

MVP 阶段可选：
- **方案A（轻量）**：使用 Vercel/Netlify Functions 或 Cloudflare Workers 作为 AI 代理
- **方案B（自建）**：Node.js Express/Fastify 服务，读取 `.env` 中的 Key，暴露 `/api/tarot/interpret` 接口
